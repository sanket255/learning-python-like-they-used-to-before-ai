# Day 7: Functions Meet Files (A Dev Diary)

---

**March 21, 2026**

Built something today that felt harder than it should've been.

Then realized I was overcomplicating it.

---

## What I Built

A note manager with functions:
- `write()` - saves notes to file
- `read()` - reads notes back
- `main()` - asks user what to do, loops

Simple concept. Messy execution at first.

---

## The Struggle

Started like this:
```python
def main():
    while date != "quit":
        queue = input("you wanna note the note: ")
        if date == "y":
            write()
        date = input("do you wanna read the note?")
        if date == "y":
            read()
```

Using `date` for everything. Checking wrong variables. Logic was tangled.

Got frustrated. Wanted to quit.

---

## The Fix

Realized: I was overcomplicating the flow.

Simplified to:
```python
def main():
    ask = 0
    while ask != "quit":
        ask = input("write(w) or read(r) or quit: ")
        if ask == "s" or ask == "w":
            write()
        elif ask == "r":
            read()
        elif ask == "quit":
            break
        ask = input("Another note or quit? (y/quit): ")

main()
```

One variable. Clear flow. Works.

---

## The Real Learning

Combining two concepts isn't harder than one concept.

It's just about keeping the logic clean.

- `write()` handles saving
- `read()` handles reading
- `main()` orchestrates (decides what to call)

Each function does ONE thing. Main calls them.

That's the pattern.

---

## The Bug I Didn't See

```python
if ask == "s" or "w":
```

This always evaluates to True because `"w"` is truthy.

Should be:
```python
if ask == "s" or ask == "w":
```

Small syntax mistake. Big logic error.

Caught it in review, not during coding.
